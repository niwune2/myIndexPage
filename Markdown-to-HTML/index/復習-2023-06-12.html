既存の部分から新しい文字列を作成する

最後の課題では、北イングランドの駅に関する情報が入っている文字列を格納している配列を扱います。この文字列にはデータ項目として、

  1. アルファベット
  2. 3文字の駅コード、
  3. 次にコンピューター読み取り用のデータ、
  4. 次にセミコロン、
  5. そして人間が読める駅名が入っています。

例:MAN675847583748sjt567654;Manchester Piccadilly

駅コードと駅名を抽出し、それを文字列にまとめてこのような形にします:

MAN: Manchester Piccadilly

このようにプログラムしてみましょう:

  1. アルファベット 3文字のコードを抽出し、新しい変数に格納します。
  2. セミコロンの文字インデックス番号を検索します。
  3. セミコロンの文字インデックス番号を参照位置にして、人が読める駅名を抽出し、新しい変数に格納します。
  4. この 2 つの新しい変数と文字列リテラルとを連結し、最終的な文字列を作成します。
  5. input 変数ではなく、この最終結果がresult 変数の値に代入されるよう修正します。
const list = document.querySelector('.output ul');
list.innerHTML = '';
let stations = 
['MAN675847583748sjt567654;Manchester Piccadilly',
'GNF576746573fhdg4737dh4;Greenfield',
'LIV5hg65hd737456236dch46dg4;Liverpool Lime Street',
'SYB4f65hf75f736463;Stalybridge',
'HUD5767ghtyfyr4536dh45dg45dg3;Huddersfield'];

for (let i = 0; i < stations.length; i++) {
    let input = stations[i];
    // この下にコードを書いてください
    let codes = input.slice(0,3);
    let semiCol = input.indexOf(';');
    let stationName = input.slice(semiCol + 1);
    let result = codes+' : '+stationName;

    let listItem = document.createElement('li');
    listItem.textContent = result;
    list.appendChild(listItem);
}
/*出力結果
・MAN : Manchester Piccadilly
・GNF : Greenfield
・LIV : Liverpool Lime Street
・SYB : Stalybridge
・HUD : Huddersfield
*/

この中で、// Add your code here以下にあるdrawCircle(x, y, size)があるのは 「円弧を描画するdrawCircle関数に、キーを押したらそれを動かす関数を適用するため」にあるという理解で合っていますか？

はい、その理解は正しいです。drawCircle(x, y, size)は円弧を描画するための関数ですが、document.addEventListener('keydown', ...)の中で呼び出されています。つまり、キーボードのキーが押されたときに実行される関数内で、drawCircle(x, y, size)が呼び出されることで円弧の位置が更新され、新しい位置で再描画されます。

なお、drawCircle(x, y, size)自体は初めに一度呼び出されており、初期位置で円弧を描画しています。その後、キーが押されるたびにdrawCircle(x, y, size)が再度呼び出され、円弧が更新されます。

このように、キーボードのイベントと円弧の描画を組み合わせることで、キー入力に応じて円弧を動かすことができるようになります。

キーを押すたびに描画を繰り返しているということですね

はい、その通りです。キーボードのキーが押されるたびに、drawCircle(x, y, size)が呼び出されて円弧の描画が繰り返されます。これにより、円弧の位置が変化し、新しい位置で再描画されるため、円弧が動くように見えるのです。

キーが押されるたびに描画を繰り返すことで、アニメーションのような効果を作り出すことができます。この場合、キーボードのキーに対するイベントがトリガーとなって描画が更新されるため、円弧の位置がキー入力に応じて動くようになります。

event.targetについて(GPT)

event.targetは、イベントが発生した要素（要素ノード）を参照するためのプロパティです。イベントがトリガーされた要素を特定するために使用されます。

イベントは通常、要素内で発生し、イベントリスナーによって処理されます。例えば、クリックイベントがボタン要素で発生した場合、event.targetはそのボタン要素を参照します。

以下は、クリックイベントが発生した要素を取得する例です。

<button id="myButton">Click me</button>

<script>
const button = document.querySelector('#myButton');

button.addEventListener('click', event => {
  console.log(event.target); // クリックされた要素を出力
});
</script>

上記の例では、ボタン要素がクリックされたときに、イベントリスナーが実行されます。そして、event.targetを使用してクリックされた要素を取得し、コンソールに出力しています。

event.targetは、イベントが発生した要素を特定するためによく使用されます。このプロパティを使うことで、イベントが発生した要素に対して特定の処理を行ったり、要素の属性や内容にアクセスしたりすることができます。

.tagnameについて(GPT)

tagNameは、要素ノードのタグ名を表すプロパティです。このプロパティは、要素の開始タグによって指定された要素の名前を取得するために使用されます。

tagNameは、大文字と小文字を区別せずにタグ名を返します。つまり、HTMLでは大文字と小文字の区別はありませんが、tagNameは常に大文字で返されます。

以下は、tagNameを使用して要素のタグ名を取得する例です。

<div id="myDiv">Hello, World!</div>

<script>
const div = document.querySelector('#myDiv');

console.log(div.tagName); // "DIV" を出力
</script>

上記の例では、div要素のtagNameを取得し、コンソールに出力しています。結果として、"DIV"という文字列が表示されます。

tagNameは要素ノードにのみ存在し、テキストノードやコメントノードなどの他のノードタイプには存在しません。そのため、要素ノードでない場合はundefinedが返されます。

注意点として、tagNameは読み取り専用のプロパティであり、タグ名を変更することはできません。

.getAttributeについて(GPT)

.getAttributeは、指定した要素の属性の値を取得するためのメソッドです。要素の属性とは、HTML要素に追加された属性（例: id、class、data-*属性など）を指します。

.getAttributeの基本的な構文は以下の通りです：

element.getAttribute(attributeName)

ここで、elementは対象となるHTML要素を表し、attributeNameは取得したい属性の名前を指定します。

例えば、次のようなHTML要素があるとします：

<button id="myButton" class="btn" data-color="red">Click me</button>

この要素からid、class、data-color属性の値を取得する場合、以下のようにgetAttributeを使用します：

const button = document.querySelector('button');

const idValue = button.getAttribute('id');
console.log(idValue); // 出力: "myButton"

const classValue = button.getAttribute('class');
console.log(classValue); // 出力: "btn"

const colorValue = button.getAttribute('data-color');
console.log(colorValue); // 出力: "red"

.getAttributeは、要素の属性の値を文字列として取得します。存在しない属性の場合はnullを返します。

なお、data-*属性にアクセスする場合は、data-の接頭辞を省略して属性名のキャメルケース形式で指定することが一般的です。

ここでは.getAttributeの基本的な使い方を説明しましたが、もちろん他にもさまざまな応用があります。必要に応じて属性の値を取得する際に活用してください。
